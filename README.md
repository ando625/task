

# タスク進捗管理アプリ

Laravel (Sanctum) と React を連携させた、SPA（シングルページアプリケーション）構成のタスク管理システムです。

## プロジェクトの概要

このシステムは、一般ユーザーが自身のアカウントを作成し、個別のタスクを管理（作成・編集・削除・ステータス更新）できるアプリケーションです。

https://github.com/user-attachments/assets/04a19d85-89fc-47eb-b29d-f320c70729fd



### 開発の背景

「チームで仕事をする際、誰が何をしているか一目でわかり、かつ自分の仕事に集中できる環境」をデジタルで再現したいと考え開発しました。
これまでのアプリ開発を経て、よりモダンな構成である**「SPA（React × Laravel Sanctum）」**に挑戦し、ページ遷移のない滑らかな操作感と、Cookieベースの安全な認証システムの構築を目指しました。

### こだわったポイント

1. **「自分」と「チーム」を切り分ける認可ロジック**
* 画面上では、自分のタスクには「編集・削除」ボタンを出し、他人のタスクには出さないという視覚的な区別を徹底しました。
* 見た目だけでなく、**「Laravel Policy」**をバックエンドに実装。万が一、他人のタスクを操作しようとする不正なリクエストが届いても、魔法の盾で守るように確実にブロックする堅牢な設計にしました。


2. **進捗を可視化する「4つのステータス管理」**
* 「未着手・進行中・確認待ち・完了」という実務に即した4段階の列を設けました。
* タスクの状態（ステータス）をワンクリックで更新できるようにし、チーム全体の「今どこで仕事が止まっているか」を直感的に把握できるようにしました。


3. **フロントとバックの「型」の連携**
* React(TypeScript)とLaravelの間でやり取りするデータの形（型）を定義し、通信時のエラーを未然に防ぐ、安全で論理的なコーディングを意識しました。



### 苦労した点

* **SanctumによるSPA認証の壁**
* サーバー（Laravel）とフロント（React）が分かれているため、どうやって「ログイン状態」を安全に維持するか（Cookieのやり取り）の理解に最も苦労しました。
* 試行錯誤の末、セッションベースの安全な認証を自力で実装できたことで、Webアプリのセキュリティ構造への理解が深まりました。


* **Reactでの状態管理（State）**
* タスクを追加・更新した際に、ページをリロードせずに画面の一部だけを書き換える「SPAならではの動き」を作るのが難しかったですが、ユーザーにとっての心地よさを追求しました。

## システム構成図（SPAの仕組み）

### 権限による操作範囲のルール

| アクション | 自分のタスク | 他人のタスク | 理由 |
| --- | --- | --- | --- |
| **閲覧** | ✅ | ✅ | チームの状況を知るため |
| **新規作成** | ✅ | - | 自分の仕事を増やすため |
| **内容の編集・削除** | ✅ | ❌ | データの改ざんを防ぐため |
| **ステータスの更新** | ✅ | ❌ | 本人以外が勝手に進捗を変えないため |

### 実装済み・実装予定の機能

### 1. 認証・認可 (Auth)

* **FN001/003 登録・ログイン**:
* **Laravel Sanctum (Cookie認証)** を採用。
* CSRF保護を有効にし、セッションベースで安全にログイン状態を管理。


* **FN005 ログアウト**:
* サーバー側のセッション破棄と同時に、React側の状態（State）をクリアし、安全に終了。

* **認可ロジック (Policy)**:
* **自分専用**: 自分のタスクのみ「編集・削除・ステータス更新」のフル操作が可能。
* **チーム共有**: 他人のタスクは「閲覧のみ」に制限。
* **バックエンド防御**: フロントエンドでのボタン非表示に加え、API側でも **Laravel Policy** を用いて、不正なリクエスト（他人のタスク操作）を確実にブロック。





### 2. タスク管理 (Tasks)

* **FN008 一覧表示**:
* 自分のタスクと他人のタスクを視覚的に区別（自分には編集・削除ボタンを表示）。
* 「全員のタスク」と「自分のタスク」を切り替えるフィルタ機能。


* **FN009~012 CRUD操作**:
* タスクの新規登録・編集・削除。
* ステータス（やること・進行中・レビュー/確認待ち・完了）のワンクリック更新。





### 3. バリデーション (Validation)

* **Backend**: Laravel `FormRequest` による厳格な入力チェック。
* **Frontend**: TypeScript の型定義を用いた安全なデータ通信。
* **ルール**: タイトル必須、詳細は255文字以内、ステータス限定（todo, doing, review, done）。

#### ## セットアップ方法 (Setup)

このプロジェクトをローカル環境で起動するための手順です。

1. **リポジトリをクローンする**
```bash
git clone git@github.com:ando625/task.git taskapp
cd taskapp

```

2. **Dockerコンテナの起動**
```bash
docker compose up -d --build

```



3. プロジェクトのルートphp上で実行
```
docker compose exec php bash
```



4. **環境設定ファイルの準備**
```bash
cp .env.example .env

```

※ .env ファイルを開き、データベースの設定を以下のように書き換えてください：

```
DB_CONNECTION=mysql
DB_HOST=mysql
DB_PORT=3306
DB_DATABASE=laravel_db
DB_USERNAME=laravel_user
DB_PASSWORD=laravel_pass
```


5. **ライブラリのインストール (PHP & JavaScript)**

```bash
composer install
```

```
npm install
```
エラーが出た場合はこちらで
```
npm install --legacy-peer-deps
```


6. **アプリケーションキーの生成とデータベースの準備**
```bash
php artisan key:generate
php artisan migrate:fresh --seed
```




7. **フロントエンドのビルド（Viteの起動）**
```bash
npm run dev

```


8. **ブラウザで確認**
- URL: http://localhost/ にアクセスしてください。


- ログインURL: http://localhost/login
- 新規会員登録URL: http://localhost/register
